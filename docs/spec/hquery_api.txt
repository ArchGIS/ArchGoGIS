Набросок спецификации для API запросов /db/hquery

Ресурсы:
/db/hquery/upsert -- добавление или редактирование данных
/db/hquery/delete -- удаление записей

## Upsert
Любой параметр имеет синтаксис:
[key] <DESCRIPTOR>:<ENTITY>
[val] <JSON_ENCODED_STRING>
где
<DESCRIPTION> - идентификатор сущности. [a-zA-Z][a-zA-Z0-9]*
<ENTITY> - категория объекта, который требуется отредактировать или создать
<JSON_ENCODED_STRING> - данные, которые относятся к объекту
пример:
'a:Monument' => '{"id": "1", "x/num": "4.5"}'

Внутри <JSON_ENCODED_STRING> находятся свойства объекта.
Если среди свойств есть "id", то произойдёт update операция,
иначе - insert. Таким образом пример выше обновит Monument,
id которого равен 1, переписывая значение x на 4.5.
У параметра id не должно быть подсказки типа, все остальные
свойства обязаны иметь один из типов:
* num -- целое или дробное число
* text -- строка произвольной длины
Важно: каждое свойство, вне зависимости от своего типа, должно
иметь строковой тип.

Пример корректного запроса из JavaScript:
```
$.post('.../hquery/upsert', {
  'monument1:Monument': JSON.stringify({
    'id': '1',
    'x/num': '10.5'
  }),
  'monument2:Monument': JSON.stringify({
    'x/num': '54.7',
    'y/num': '14.0',
    'note/text': 'I like this one'
  })
});
```
В результате будет обновлена 1 запись, а другая будет создана.
